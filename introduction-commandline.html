Using Gregorio in a terminal

This page is an introduction to working with gregorio from the command line. This page is reserved for users who are used to compile TeX documents in command line. If you are not, please start with an editor.

The command line tool gregorio converts a score written in gabc into a TeX file. GregorioTeX provides mainly the \includescore that takes the filename of your gabc score and includes it. It automates gabc to TeX compilation by automatically executing the gregorio command when needed. See UserManual.pdf for more details.

Gregorio extensively uses the new features of LuaTeX, so compilations needs to occur with this engine. Also, in order to compile your scores automatically, lualatex needs to be able to run external commands. To allow it, run lualatex with the --shell-escape argument, or add gregorio to the list of allowed commands by restricted command execution (the list is named shell_escape_commands in texmf.cnf).

Behind the scenes, the automatic compilation creates foo-bar.gtex where foo is the base name of your gabc score and bar is the version of Gregorio.
