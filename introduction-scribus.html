This page describes the use of gregorio inside the free DTP software called Scribus. This possibility works from version 1.0 of gregorio and version 1.3.5 of Scribus.

Scribus

Scribus is a free DTP software that has this advantage to have a simple and accessible graphical user interface. It is possible to include gregorio scores in a Scribus document, and thus to build simply a book or a booklet containing gregorian scores.

A page describing the process to have the latest version of gregorio under Linux is available on the Scribus web site. The Windows version is available on the download page and the instructions for Mac here. Note that in order for the gregorio score import to work, you also need ghostscript, certainly already installed under Linux, and available for Windows on the sourceforge page.

If you encounter problems with the Scribus software outside of its interaction with gregorio, you can consult the Scribus documentation.

Installation of the additional Scribus files

To benefit from the import of the gregorio scores, some additional files for Scribus must be installed. These files are available in the contrib/ directory of the gregorio sources, and also in the subversion repository. These files are gregorio-scribus.lua, 900_gregorio.xml and gregorio.png. If you are under Windows, it's better to take them that have been installed with the automatic installer, they are in C:\Program Files\Gregorio\contrib (assuming you have gregorio installed here).

If you are under Mac or if you have installed Gregorio under Windows with the advanced method, you need to slightly modify these files to adapt them to your system. These modifications are described in the comments of the file 900_gregorio.xml.

Finally, you need to make Scribus use these files. To do so go in File/Preferences, and then, in the External Tools part, in Render Frames, click on add and select the file 900_gregorio.xml in the contrib/ directory of the gregorio installation. An interesting trick if you use only gregorio score import, is to make gregorio the first of the list, in order to use it by default. To do so click on Up several times. Click on Apply then OK, and it's done.

Use in Scribus

The use fo gregorio inside Scribus is almost the same as the LaTeX one: click on Insert/Insert Render Frame, then select the area in which you want a score to appear. Right-click on this area and click on Edit Source, and the editor will open.

First select gregorio in the program list (if it's not by default). The left text area allows you to enter the gabc code of the score. Two other tabs are present: one to add some packages to the LaTeX header, and one to put some GregorioTeX commands.

The rendering may be scary at first, but in fact it's only the preview that has a low quality, which is not the case of the final rendering; which you will see if you export in PDF.

When you will try to export a PDF, scribus will output some warnings, you can ignore them safely. Also you will need to check the box Embed EPS & PDF files (EXPERIMENTAL). If you cannot check this box (it happens on certain versions, especially under Linux), the scores will be included as images, which can introduce pixelization. To improve the quality of the images, you can change the value of the option Resolution for EPS Graphics; 600dpi is a standard value for desktop printers. Note that in this case the produced PDF will be as much as 10 times heavier.
