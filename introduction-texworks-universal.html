This page describes how to configure gregorio for use with TeXworks on a Linux box. You will have to repeat this process for each user who uses TeXworks.

Configure TeXworks

While your screens may look a little different depending on your operating system, the steps to configure TeXworks are the same whether you're on Linux, Mac, or Windows.

All we need to do is add the compilation engine for LuaLatex to TeXworks. To do so, go in Edit -> Preferences(TeXworks -> Preferences on a Mac) and then in the "Typesetting" tab, in the "Processing tools" section, click on add ("+"), and enter the following data:


Adding LuaLaTex in TeXworks

If you want, you can also add LilyPond-Book in a similar fashion (the "--shell-escape" argument is not necessary), though you'll first need to get the engine from its maker.

Once you've downloaded the engine, open TeXworks and click on Help -> Settings and Resources... Follow the link to the Resources folder in the pop-up dialog that appears. In this folder create a new subfolder called "engines" and copy it there. Note: This engine does not work on Windows.

If everything goes as planned, then you're ready to start working with TeXworks. If not, see below.

Troubleshooting

The most common problem to run into at this stage is for TeXworks to return the following error when you try to add an engine:


Problem!

This problem occurs because some systems automaticaly restrict you from running downloaded files as a saftey precaution. To let your system know that it's okay to run this file we have to set the executable bit.

To do this start by opening Terminal (it's in Applications/Utilities) then enter the following commands (you can leave out the lines associated with engines you're not setting up):

cd ~/.TeXworks/engines
chmod a+x gregorio.engine
chmod a+x greg-book.engine
chmod a+x greg-lily-book.engine
chmod a+x LilyPond-Book.engine

Once you've done this, you should be able to add the engines to TeXworks as described above.

If you're still having trouble, then sign up for the users mailing list (instructions are here) and send an email describing your problem in as much detail as possible. Someone will help you get your issue resolved.
