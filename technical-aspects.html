Source code

The source code of Gregorio is entirely under GNU GPL and is hosted on a subversion server of gna.org.

You can browse the code online or get a local copy of the repository by checking the address svn://svn.gna.org/svn/gregorio/trunk out.

Internals

The gregorio software has a very modular structure that allows developers to very easily add read or write modules for a format type (OpusTeX, *XML, ...).

The program itself is written in C. Some parts (in particular modules to read gabc) are written in flex and bison. The TeX part is written in Omega, for the additional primitives it offers, such as, for example, adding a box at the end or at the beginning of a line (with \localeleftbox{}). This makes possible the automated creation of the custos or clef for each line without manually specifying the end of line. The gregorio font (for GregorioTeX) was designed with fontforge.

gregorio has an internal structure (built with struct in C) to represent a score in memory. This structure is based on nested linked lists (score → syllable → element → glyph → note). The reading functions transform a variable of type 'file' into that structure and the writing functions transform this structure into another variable of type 'file'. This modularity makes it possible to write a reading or writing function independently of the rest of the program.

The C data structure has a structure very close to that of GregorioXML.

Future work

There is still much to develop in Gregorio. Here is a quick overview.

Additional functions

Some functions are still missing in Gregorio, notably:

  • export in lilypond format, even though the Gregorian chant part of lilypond in not very good and not maintained anymore
  • export in MusiXTeX format, though for similar reasons it is not urgent
  • importing from OpusTeX, because it is the most used format for the moment

If you are interested in the development of one of these functions, do not hesitate to contact the Gregorio developers, even if you are not a C guru.

A graphical interface

It is planned to make a graphical interface for Gregorio, in Python/Qt, so that anyone will be able to create his own Gregorian chant scores as simply as possible.

In this graphical interface, we would type gabc notation and the score would appear in real time on the screen, thanks to glyphs of the gregorio font converted into SVG format.

This GUI is already started and has advanced, but current development is focusing on GregorioTeX and code stabilization.
