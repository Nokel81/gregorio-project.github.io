This page describes the installation and compilation of gregorio software under a GNU/Linux system. Whether you install a precompiled version or whether you compile from source, it's a quick and easy operation.

On Debian and Ubuntu systems

To obtain the sources of the stable version, you just need to download them from the download page, and decompress the archive.

To get the SVN version of the source code, that is, the version in development, you will have to create a local copy of the subversion repository of the project, with the command:

svn co svn://svn.gna.org/svn/gregorio/trunk
cd trunk

Gregorio's software includes compilation tools specific to debian systems (including Ubuntu), for both the program gregorio and the TeX style GregorioTeX , as debian's installation method differs from that of other distributions.

In order to compile gregorio from source on debian, it's necessary to install several packages:

sudo aptitude install libxml2-dev flex bison docbook-to-man xsltproc autotools-dev automake libtool libltdl-dev tex-common docbook-xsl autopoint python-fontforge fakeroot dh-autoreconf

Then, to construct the debian packages corresponding to gregorio and GregorioTeX, type:

dpkg-buildpackage -rfakeroot -d -b

This will produce two debian packages. You can find them in the parent directory, which you will be able to install as follows:

sudo dpkg -i ../gregorio*.deb

For other GNU/Linux systems

On other systems, you will need to compile gregorio. In order to do this, you will have to install the software packages necessary for compilation, which are gcc, automake, libltdl-dev, libtool, libxml2, libxml2-dev, flex, bison, python, autoconf, texlive and make. (Note: on some distros, packages may be called foo-devel instead of foo-dev).

To obtain the sources of the stable version, you can download them from the download page, and decompress the archive.

OR: To get the latest version of the source code, that is, the version in development, you will have to create a local copy of the git repository of the project. Use these commands:

git clone https://github.com/gregorio-project/gregorio.git gregorio-git
cd gregorio-git

Once those preparatory steps are done, simply execute the following commands:

$ autoreconf -f -i (if you have the git version)
$ ./configure --prefix=/usr
$ make
$ make install
$ cd fonts
$ sudo make install

Update

To update your git version, you just need to go in the directory containing the source files (assuming you kept them), run git pull origin master and recompile with the same instructions as the previous Compilation section. If you deleted that directory after install, then just repeat the steps above starting at the git clone command.
